
# 探索方法の説明

## 離散化

ロボットの歩行をグラフ探索で制御するための方法を説明する．
まず，ロボットの状態を次のように[(int, bool)]のタプル配列で表現する．

- int: ロボットの現在の脚位置を表す整数
- bool: ロボットが地面に接地しているかどうかを表す真偽値

int はロボットの脚位置を離散的なステップで表す．4は基準位置，1~3は後ろ側の位置，5~7は前側の位置を示す．
また，1,5は下，2,4,6は中間，3,7は上側の位置を示す．

boolはtrueの場合，ロボットが地面に接地していることを示し，falseの場合は遊脚にあることを示す．

## グラフの遷移

ロボットの状態遷移は以下のルールに従う．

1, 支持脚の切り替えの遷移:
    - [(4,T), (4,F), (4,T), (4,F), (4,T), (4,F)] -> [(4,F), (4,T), (4,F), (4,T), (4,F), (4,T)]
    - 接地している脚を遊脚に上げる動作，あるいは遊脚を接地させる動作を表す．
    - intを変更せず，boolを反転させる．

2, 脚位置の移動の遷移:
    - [(4,T), (4,F), (4,T), (4,F), (4,T), (4,F)] -> [(4,T), (6,F), (4,T), (6,F), (4,T), (6,F)]
    - 遊脚を前方または後方に移動させる動作を表す．
    - 接地している脚の位置(int)を変更せず，遊脚の位置(int)を変更する.

これらを交互に適用することで，ロボットは歩行動作を実現する．

# 頻度分析

今回は，状態遷移の繰り返しによって生成される状態列に対して，頻度分析を行う方法を説明する．
なお，1. 2. の順番で探索するため，この(1. 2.)を1セットとしてカウントする．

## 頻度分析の手順

1. 状態列の生成:
    - 初期状態から始めて，1. 支持脚の切り替えと2. 脚位置の移動を交互に適用し，状態列を生成する．
    - 例えば，初期状態[(4,T), (4,F), (4,T), (4,F), (4,T), (4,F)]から始めて，複数のステップを経て状態列を作成する．
2. 状態のカウント:
    - 生成された状態列に対して，各状態の出現回数をカウントする．
    - 状態は[(int, bool)]のタプル配列として表されるため，各タプル配列をキーとして使用し，その出現回数を値として保持する辞書を作成する．
3. 頻度の計算:
    - 各状態の出現回数を総ステップ数で割ることで，各状態の頻度を計算する．
    - 頻度は，状態が全体の中でどれだけ頻繁に出現するかを示す指標となる．
4. 結果の表示:
    - 各状態とその頻度を表示する．
    - 頻度が高い状態は，ロボットの歩行動作において重要な役割を果たしている可能性がある．
